## AC-Test Mapping Rules

1. **When writing spec**: spec produced by `/plan` automatically includes an AC-Test Mapping table with each AC defaulting to ⬜
2. **During development**: remind to update the corresponding Status when each AC implementation is complete
3. **When preparing to commit**: if any AC is still ⬜, ask "AC-{N} doesn't have a corresponding test yet — add one now?"

### When AC-Test Mapping is not needed

- Pure documentation updates (README, CHANGELOG)
- Pure style changes (CSS-only changes, no logic)
- Dependency updates (version bumps)
- Hotfix (but a regression test should still be added)
